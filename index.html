Plain-English lesson

A good cryptographic hash behaves as if the output has been scrambled by a black box.

3What is the lesson?

A SHA-256 hash is a fixed-length fingerprint for text or data. It always gives 64 hexadecimal characters, whether the input is one letter or a whole book.

  • Same input: same hash every time.
  • Tiny change: usually about half the bits flip.
  • One-way: the hash does not tell you the original message.

Real-world uses

  • File check: “Is this download exactly the file the website promised?”
  • Tamper evidence: “Did someone silently edit this record?”
  • Blockchains: “Can a computer find a hash with a rare starting pattern?”
  • Passwords: real systems use salted, deliberately slow password hashing; plain SHA-256 alone is not enough.

Bonus demo: hash lottery

This is optional. It repeatedly adds a number, called a nonce, until the fingerprint starts with zeros. That is the basic “try many guesses” idea behind proof-of-work, shown here as a tiny browser-safe lottery.

Optional: press “Find lucky hash” to make the browser search for a hash beginning with zeros.

Not encryption

Encryption is for hiding a message and later unlocking it. Hashing is different: it makes a fingerprint you can compare. You do not decrypt a SHA-256 hash.
